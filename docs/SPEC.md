## 1. App Structure

### 1.1 Architecture: Orb Home Hub (NOT Linear)

The app is a circular hub, not a linear slideshow. Every session starts and ends at Orb Home.

```
First launch → Onboarding (4 steps, one-time) → Orb Home

Every subsequent launch → Orb Home
  ↓ (tap Orb)
  Moment 1: Catch You
  ↓ (swipe right on restaurant)
  Moment 2: Go With You (ONE scrollable screen)
  ↓ (scroll to bottom, indicate arrival)
  Moment 3: Remember Us (ONE conversation)
  ↓
  Back to Orb Home (new memory bubble added)
```

### 1.2 Structural Rules

1. **Orb Home is the hub.** Every session starts here, every journey ends here.
2. **Moment 2 is ONE scrollable screen.** Story + dishes + navigation in one continuous scroll. No fake walking animation.
3. **Moment 3 is ONE conversation.** Feedback + optional photo in one Orb dialogue. No separate share screen.
4. **After Moment 3, return to Orb Home** with a new memory bubble. NEVER "restart." NEVER dead-end.
5. **Daily use = Orb Home → 3-4 interaction beats → Orb Home.** Not 13 screens.
6. **Settings:** gear icon on Orb Home, opens as overlay.

## 2. Orb Behavior Rules

### 2.1 Two Modes

**Conversation mode (center, large):**
- When: Gusto is talking to user or waiting for response
- Behavior: Orb at screen center. Text appears above orb. When user needs to respond → Orb fades out → bubble buttons emerge from screen center → user taps → bubbles collapse → Orb fades back in.
- Visual focus always at screen center.

**Display mode (top, tiny dot):**
- When: Gusto is showing content — cards, restaurant info, dishes
- Behavior: Orb retreats to top as tiny breathing dot (~16-20px). Content fills screen below.
- User interacts with content (swipe cards, scroll), not with Orb.

### 2.2 Mode Switching Rules

| From → To | Trigger |
|-----------|---------|
| Conversation → Display | Orb finishes dialogue, needs to show content (cards, scroll) |
| Display → Conversation | User needs to respond to a question; OR swipe-left/swipe-down triggers Orb return; OR arrived check-in |

Transition: Orb smoothly moves + scales between positions (400ms). Never jumps.

### 2.3 Orb States

| State | Visual | When |
|-------|--------|------|
| Resting | Slow breathing pulse | Idle on Orb Home, waiting |
| Greeting | Resting + text above | Moment 1 Beat 1 |
| Thinking | Faster tremor + shimmer | After user swipes left, before showing next card |
| Absorbing | Brief flash | After receiving user feedback |
| Fading out | Opacity → 0 | Before bubbles emerge |
| Fading in | Opacity 0 → 1 | After bubbles collapse |

## 3. Interaction Patterns

### 3.1 Bubble Button Interaction

**When used:** All dialogue responses — onboarding choices, mood selection, arrival feelings, meal feedback. Anything where Gusto asks and user responds.

**Flow:**
1. Gusto's text visible above Orb
2. 1.5-second pause (user reads, feels greeted — CRITICAL, do not skip)
3. Orb fades out (320ms)
4. 2-3 circular bubbles emerge from screen center (spring animation, staggered)
5. User taps one bubble
6. Selected bubble: shrinks to center (320ms)
7. Unselected bubbles: scatter outward in different directions (450ms)
8. Orb fades back in (320ms)
9. Gusto responds (200ms pause, then text fades in)

**Content rule:** ALL bubble options must be contextual — generated by LLM based on time, location, user history, current moment. Never hardcoded. The prototype may use representative examples, but production must generate dynamically.

### 3.2 Card Swipe Interaction

**When used:** All accept/reject choices in display mode — taste test photos, restaurant recommendations, dish recommendations, AI summary confirmation.

**Gesture rules:**
- **Swipe right = Accept.** Proceed to next step. Don't ask why. Respect the decision.
- **Swipe left = Reject.** Orb returns to center, "Let me think..." → show next card. Don't ask why.
- **Swipe down = Change direction.** Orb returns to center, asks a light question → contextual bubbles emerge. Only path that asks follow-up.

**Critical rule: No consecutive swiping.** After every swipe, Orb must return to center briefly to "think" before showing next card. Exception: Visual Taste Test allows faster rhythm (~100ms pause) because it's quick preference scanning.

**Difference from Tinder:** Tinder = swipe → next → swipe → next (infinite loop). Gusto = Orb dialogue → show card → swipe → Orb thinks → show next. Always a "friend thinking" moment between cards.

### 3.3 Scrollable Content

**When used:** Moment 2 (restaurant story + dishes + navigate).

**Behavior:** Standard vertical scroll. Orb stays as tiny dot at top throughout. No interaction interruptions during scroll — user reads at their own pace. Interaction resumes at bottom (arrival check-in).

## 4. Screen-by-Screen Flow

### ONBOARDING (first launch only)

**O1: Welcome**
- Mode: Conversation
- Content: Orb center + "Hey, I'm Gusto." above + "Tap me to get started" below
- Action: Tap orb → transition to O2
- Notes: Nothing else on screen. No food photos. No feature list.

**O2: Exclusions**
- Mode: Conversation
- Content: "By the way, anything you can't eat?" above Orb
- Action: Orb fades out → 2 bubbles emerge ("I eat everything" / "Some things to avoid")
- Path A (eat everything): bubbles drift away → Orb returns → transition to O3
- Path B (things to avoid): more bubbles emerge (allergens, religious, vegan). User taps to select (multi-select). Selected glow + bounce. When done → bubbles return to center → Orb returns → transition to O3

**O3: Visual Taste Test**
- Mode: Display (Orb at top as dot)
- Content: "Swipe right if it makes you hungry" + food photo cards, one at a time
- Action: Swipe right = like, swipe left = skip. 10 cards total. Progress dots.
- Notes: Faster rhythm than restaurant cards (~100ms Orb pause). After last card → transition to O4.

**O4: AI Summary**
- Mode: Display (Orb stays as dot at top)
- Content: "I think I've got you." + summary card with preference text
- Action: Swipe right = confirm → Orb flashes → transition to Orb Home (first time). Swipe left = adjust → Orb returns to center, bubbles emerge for correction.

### ORB HOME (hub — every session starts and ends here)

- Mode: Conversation (Orb at center, resting)
- Content:
  - Orb center, breathing slowly
  - Memory bubbles floating around Orb (3-7, organic scatter)
    - Each = past dining experience (tiny photo or initial, 24-36px)
    - Gentle floating animation
  - First-time (no memories): Orb alone + "tap me when you're hungry"
  - Settings: small gear icon, top-right corner
- Actions:
  - Tap Orb → Enter Moment 1
  - Tap memory bubble → Brief flashback card (restaurant, date, Gusto's comment)
  - Swipe up → Memory timeline (scrollable list of past journeys)
  - Tap gear → Settings overlay

### MOMENT 1: CATCH YOU

**Beat 1 — Greeting**
- Mode: Conversation
- Trigger: User tapped Orb on Home
- Content: Context-aware greeting fades in above Orb. Examples:
  - Weekday evening: "Hey, long day today?"
  - Weekend noon: "Weekend vibes — hungry?"
  - Late night: "Late night? I know a place."
- **CRITICAL: 1.5-second pause after greeting before showing bubbles.** Let user feel greeted, not interrogated.
- Then: Orb fades out → bubbles emerge

**Beat 2 — User responds**
- Mode: Conversation (bubbles visible, Orb faded out)
- Content: 2-3 contextual bubble buttons. Examples:
  - Evening: "Yeah, something warm" / "Feeling adventurous" / "You decide"
  - Weekend: "Brunch vibes" / "Something new" / "Quick bite"
- Action: User taps → bubbles collapse → Orb returns → Orb "thinking" state

**Beat 3 — Recommendation**
- Mode: Display (Orb shrinks to top dot)
- Content: "I've got just the place." + restaurant card slides up
  - Card contains: food photo, name, cuisine, distance, ambiance tag, one-line reason
- Action:
  - Swipe right → proceed to Moment 2
  - Swipe left → Orb returns to center, "Let me think..." → new card
  - Swipe down → Orb returns to center, "What direction?" → contextual bubbles

### MOMENT 2: GO WITH YOU (ONE scrollable screen)

- Mode: Display throughout (Orb as dot at top)
- Content: Single vertical scroll containing:

**Section A — Restaurant story**
- Restaurant exterior/interior photo (contextual)
- "Let me tell you about this place..."
- 1-2 sentences of story
- Neighborhood note
- Ambiance description

**Section B — What to order**
- "Here's what I'd suggest:"
- 2-3 dish cards in HORIZONTAL scroll (not one-at-a-time swipe)
  - Each: dish photo, name, price, one-line reason
  - Source badge: "✓ on their menu" or "◊ seen in reviews"
  - Tap to "add" (card gets terracotta border glow)
- Below: "Based on what I found online — check the menu when you arrive"
- Adjustment: dashed-border button "menu doesn't have these" → Orb returns to center, contextual bubbles

**Section C — Navigate**
- Restaurant address + "Open in Maps" button
- Gusto note: "It's a 5 minute walk — enjoy it."
- NO fake walking animation.

**Section D — Arrival check-in (bottom of scroll, transitions to Moment 3)**
- After breathing space at bottom of scroll:
- Orb dot returns to center, grows large (conversation mode)
- "You there? How does it feel?"
- Orb fades out → bubbles emerge: contextual arrival feedback options

### MOMENT 3: REMEMBER US (ONE conversation)

> All steps after initial feedback are optional micro-interactions within the same conversation. Not separate screens.

**Beat 1 — Feedback**
- Mode: Conversation
- Trigger: Arrival check-in completed
- Content: "How was it?"
- Orb fades out → 3 bubbles: "Amazing" / "Pretty good" / "Just OK"
- User taps → bubbles collapse → Orb returns
- Gusto responds contextually: "Knew you'd love it!" or "Noted, I'll find better next time."

**Beat 2 — Optional record + close**
- Mode: Conversation
- After Gusto's response: small camera bubble + "done" bubble appear near Orb (gentle, optional — not a full-screen takeover)
- If user taps camera: photo taken → appears in warm frame → "That looks incredible!"
- If user taps "done": skip photo
- Gusto: "I'll remember this one."

**Transition back to Orb Home:**
- Orb pulses warmly → screen transitions to Orb Home
- New memory bubble appears among floating bubbles
- New bubble briefly glows (terracotta tint) to indicate it just arrived
- The journey enriches the home, not restarts it.

## 5. Adaptive Rhythm

### Quick mode (hungry, rushed)
- Greeting: very short ("Hey! Found one.")
- Fewer bubbles (2 instead of 3)
- Moment 2: condensed scroll (shorter story, 2 dishes, immediate navigate)
- Moment 3: "How was it?" + one tap → home
- Animations: faster (200ms instead of 400ms)

### Comfort mode (sad, lonely)
- Greeting: longer, warmer, with pause
- More breathing space everywhere
- Moment 2: fuller story, neighborhood detail
- Moment 3: more celebratory response, photo encouraged
- Animations: slower (500ms)

### Mode detection (MVP)
- Time of day + day of week → initial guess
- User response to greeting bubble → confirms/overrides
- "Just hungry, be quick" → quick mode
- "Yeah, need comfort" → comfort mode

## 6. Data Model (what to persist)

- Hard exclusions (allergens, religious, vegan)
- Onboarding preference profile (LLM summary + raw tags)
- Ambiance preferences (learned: solo vs social, quiet vs lively)
- Interaction history (restaurant, response, timestamp, context)
- Restaurants visited
- Moment 3 records (feedback, photos)
- Adaptive rhythm tendency (quick vs comfort)

## 7. Technical Stack

| Layer | Choice | Why |
|-------|--------|-----|
| Platform | PWA (Next.js) | Fast to build, no App Store |
| Deploy | Vercel | Free, one-command |
| Storage | Supabase | Free tier, cloud persistence |
| Identity | Device ID (v1) → account (v2) | No login for MVP |
| LLM | Claude API + Web Search | Recommendations + dish info |
| Restaurant data | Google Places API (New) | Photos, reviews, location |
| Dish data | LLM + Web Search (real-time) | Better than any menu API |

- Use picsum.photos for placeholder food images
- Include the dev navigation bar at bottom for jumping between screens
- framer-motion for all animations
- Mobile viewport (375px)
- Single index.html file is fine for prototype
